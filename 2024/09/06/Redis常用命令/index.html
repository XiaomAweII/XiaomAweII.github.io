字符串(Strings)

Redis 缓存中的操作时间福再度不需要完全记忆, 不过了解常用的操作时间复杂度对于我们更好的使用 Redis 和优化性能, 记住每一部分的特性即可!
String 大多数操作时间复杂度 O(1)

SET

设置键key的值为value

SETEX

将键key的值设置为value, 并且超时时间为milliseconds毫秒

只有在键 key 不存在的情况下, 将key的值设置为value

PSETEX

将键key的值设置为value, 并且超时时间为milliseconds

GET

返回键key的值设置的value

GETSET

将键key的值设置为 value, 并返回设置前的旧值. 若没有旧值则返回nli

APPEND

如果键key存在并且值是一个字符串, 则把value追加到现有值的末尾

如果key不存在, 则将key的值设置为value

返回追加value之后该value的长度

SETRANGE

最好时间复杂度 O(1), 最坏时间复杂度 O(N)
从偏移量offset开始用value字符串覆盖键key存储的原字符串. offset从 0 开始返回修改后新的value字符串的长度

GETRANGE

时间复杂度 O(N)
返回key存储的字符串的从start到end之间(包括start和end)的部分.
返回value指定的子串

INCR/DECR

将键key存储的数字值value加一/减一.
若key不存在则值优先初始化为 0 再加一/减一.
若key存储的值不能被解释为数字,则返回错误.

INCRBY/DECRBY

将键key存储的数字值value加上/减去一个increment/decrement的量.
若key不存在则值先初始化为 0 再加/减.
若key存储的值或increment不能被解释为浮点数值,则返回错误
区别就在于原先只能加/减 1,这个可以加减一个量

INCRBYFLOAT

将键key存储的浮点数值value加上一个increment的增量.
若key不存在则值先初始化为 0 再执行加上增量的操作.
若key存储的值或increment不能被解释为浮点数值, 则返回错误.

MEST

时间复杂度 O(N)

同时为多个键设置值

MGET

时间复杂度 O(N)

返回给定的一个或多个键的值

MSETNX

时间复杂度 O(N)

当所有给定的key都不存在时才同时为多个键设置值
只要有一个key已经存在, 那么所有值都不会被继续执行

DEL

删除给定的一个或多个键的值

STRLEN

获取键位key的值的字符串长度
